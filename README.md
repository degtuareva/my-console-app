# Library Management Console App

Консольное приложение для управления библиотекой книг с применением принципов **SOLID** и объектно-ориентированного
программирования.

## 📋 Описание проекта

Приложение позволяет:

- Добавлять, искать и удалять книги
- Добавлять и управлять читателями
- Бронировать книги для читателей
- Возвращать забронированные книги
- Отображать статистику библиотеки

## 🎯 Тема приложения

**Управление библиотекой книг**

### Сущности (3+ классы):

| Сущность     | Класс                                                     | Описание                                           |
|--------------|-----------------------------------------------------------|----------------------------------------------------|
| Книга        | [`Book`](src/main/java/com/example/model/Book.java)       | Название, автор, год издания, ISBN                 |
| Читатель     | [`Reader`](src/main/java/com/example/model/Reader.java)   | Имя, ID, телефон                                   |
| Бронирование | [`Booking`](src/main/java/com/example/model/Booking.java) | Связь книги и читателя, дата бронирования/возврата |

## ✅ Принципы SOLID

Приложение реализует все 5 принципов SOLID:

### 1. **S**ingle Responsibility Principle (SRP)

Каждый класс имеет единственную ответственность:

- `Book` — только описывает книгу
- `BookDao` — только управляет данными книг
- `LibraryService` — только реализует бизнес-логику
- `AddBookCommand` — только обрабатывает команду добавления книги

### 2. **O**pen/Closed Principle (OCP)

Классы открыты для расширения, но закрыты для изменения:

- `LibraryService` можно расширить новыми методами без изменения существующего кода
- `Command` интерфейс позволяет добавлять новые команды без изменения CLI

### 3. **L**iskov Substitution Principle (LSP)

Подклассы могут заменять базовые классы:

- Все команды реализуют интерфейс `Command` и могут использоваться одинаково
- DAO-классы работают через абстракции

### 4. **I**nterface Segregation Principle (ISP)

Интерфейсы имеют минимальное количество методов:

- `Command` интерфейс имеет только 3 метода: `execute()`, `getName()`, `getDescription()`
- DAO-классы не зависят от лишних методов

### 5. **D**ependency Inversion Principle (DIP)

Зависимости инвертированы через абстракции:

- `LibraryService` зависит от DAO-интерфейсов, а не от конкретных реализаций БД
- CLI зависит от интерфейса `Command`, а не от конкретных команд
- Используется **Dependency Injection** через конструкторы

## 🛠️ Разработка

### Добавление новой команды:

1. Создайте новый класс, реализующий интерфейс `Command`
2. Добавьте команду в `CommandFactory.initializeCommands()`
3. Реализуйте методы `execute()`, `getName()`, `getDescription()`

### Пример:

```java
public class DeleteBookCommand implements Command {
    private final LibraryService libraryService;

    public DeleteBookCommand(LibraryService libraryService) {
        this.libraryService = libraryService;
    }

    @Override
    public String execute(Scanner scanner) {
        // Логика удаления книги
        return "Книга удалена";
    }

    @Override
    public String getName() {
        return "Удалить книгу";
    }

    @Override
    public String getDescription() {
        return "Удалить книгу из библиотеки";
    }
}
```
